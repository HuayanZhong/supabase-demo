# Heuristic — 前端评估阶段最佳实践

## 评估前加载顺序

```
1. workflows/frontend/{task-type}.md 完成检查清单  → 明确评估基准
2. execution-plan/frontend/（3 文件）              → 明确规划承诺
3. execution-engine/frontend/（3 文件）            → 明确执行记录
4. 读取实际产出文件                                → 对比预期
```

## 评估流程

### 新建组件/页面（create）

```
① 确认文件存在        → 是否在预期路径创建了文件
② 类型检查            → pnpm check-types
③ 代码质量            → pnpm lint
④ i18n 检查           → 翻译 key 是否在代码中使用 t() 调用
⑤ 样式检查            → 是否使用语义色而非硬编码色值
⑥ 结构检查            → 组件拆分是否合理（不超过 200 行？）
⑦ 数据流验证          → 预览确认数据能正确传递到模板，无 undefined 渲染
⑧ 多余文件检查        → apps/frontend/ 范围无未列入计划的文件
```

### 修改功能（modify）

```
① 确认修改点          → 计划修改的文件是否确实被修改
② 只读文件验证        → 计划外文件是否被误改
③ 类型检查            → pnpm check-types
④ 回归验证            → 引用方是否正常工作
⑤ 数据流验证          → 预览确认数据传递逻辑未断裂
⑥ 多余文件检查        → apps/frontend/ 范围无未列入计划的文件
```

### 修复 Bug（fix）

```
① 确认 Bug 不再复现   → 按原复现步骤操作
② 修改范围验证        → 是否只改问题区域（最小修改原则）
③ 类型检查            → pnpm check-types
④ 回归验证            → 关联功能是否正常
⑤ 注释检查            → 非显而易见逻辑是否有注释说明
⑥ 多余文件检查        → apps/frontend/ 范围无未列入计划的文件
```

### 重构（refactor）

```
① 新文件确认          → 按计划创建了哪些文件
② 旧文件确认          → 旧代码是否迁移干净，无残留
③ 类型检查            → pnpm check-types
④ 引用更新验证        → 所有引用方已更新导入路径
⑤ 多余文件检查        → apps/frontend/ 范围无未列入计划的文件
```

### 样式调整（style）

```
① 实际效果检查        → 标注验证方式（预览/截图）
② 语义色验证          → 无硬编码色值
③ 响应式检查          → 移动端→平板→桌面适配
④ 多余文件检查        → apps/frontend/ 范围无未列入计划的文件
```

### 国际化（i18n）

```
① 代码检查            → t() 调用是否在模板/script 中正确使用
② key 清单            → 新增 key 是否已记录移交清单
③ 翻译文件豁免        → 确认未直接修改翻译文件
④ 多余文件检查        → apps/frontend/ 范围无未列入计划的文件
```

## 验证命令执行顺序

```
pnpm check-types    → 类型零错误（阻断性）
pnpm lint           → error 零（非阻断性可容忍但记录）
pnpm format         → 无未格式化文件
pnpm build          → 编译通过（如适用）
```

命令按此顺序执行，前面的失败应修复后再继续后续。

## 评估证据采集

- 类型检查 → 保存命令输出片段作为证据
- lint 结果 → 记录 error 数量和类型
- 文件存在性 → 用 `Glob` 或 `LS` 确认
- 文件变更 → 用 `git diff`（如适用）确认修改范围
- 翻译 key → 搜索 `t(` 调用确认使用情况

## 回归范围建议

| 变更类型        | 建议回归范围                   |
| --------------- | ------------------------------ |
| 修改通用组件    | 搜索该组件的所有引用页面       |
| 修改 composable | 搜索该 composable 的所有调用方 |
| 修改 API 调用   | 确认请求/响应结构一致          |
| 修改类型定义    | 搜索该类型的所有引用方         |
| 新增页面        | 检查导航/路由是否正确注册      |
