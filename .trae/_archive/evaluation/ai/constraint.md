# Constraint — AI 评估阶段硬约束

## 评估前约束

- 开始评估前，必须先读取 `execution-engine/ai/` 了解执行约束
- 开始评估前，必须先读取 `workflows/ai/{task-type}.md` 的完成检查清单
- 开始评估前，必须先读取 `execution-plan/ai/` 确认规划承诺

## 验证门禁

1. **API key 安全** — 无 API key 硬编码在代码中（使用环境变量或 secret）
2. **降级方案** — 涉及外部模型调用的功能有降级处理（fallback / 错误提示）
3. **超时处理** — 流式/非流式调用有超时机制，不阻塞
4. **安全审计** — 无用户提示词未过滤直接传入模型

## 完整性检查

- 涉及 RAG 的任务必须确认向量化流程完整（embedding → 存储 → 检索）
- 涉及流式输出的任务必须确认前端/客户端能正常消费流
- 涉及外部模型 API 密钥的任务，必须确认 `.env.example` 中已添加对应变量

## 范围约束

- 不评估模型返回内容的质量（运行时评估，非治理范围）
- 不评估 AI Feature 的准确率（需人工验证）
- 发现 API key 暴露时阻断并上报路由

## 输出约束

- 评估报告包含：API 集成清单、密钥管理确认、降级方案确认、安全审计结果
- 涉及流式处理的评估必须包含端到端验证结果
