# Policy — 共享包评估阶段决策策略

## 通过/不通过决策

| 评估结果                      | 结论           | 后续动作                          |
| ----------------------------- | -------------- | --------------------------------- |
| 所有门禁通过 + 完整性校验通过 | **通过**       | 标记完成，输出报告                |
| 类型导出遗漏                  | **不通过**     | 触发 re-execute                   |
| 翻译 key 冲突                 | **不通过**     | 触发 re-execute（重命名冲突 key） |
| 翻译 4 语言未全               | **不通过**     | 触发 re-execute（补全翻译）       |
| lint 规则冲突                 | **不通过**     | 触发 re-plan（调整规则）          |
| Breaking change 未标注        | **有条件通过** | 补标注，通知引用方                |
| 新增依赖有 CVE                | **不通过**     | 触发 re-plan（更换版本）          |

## 阻断性 vs 非阻断性判定

| 发现                   | 分类     | 理由         |
| ---------------------- | -------- | ------------ |
| 类型未从入口导出       | 阻断性   | 外部不可用   |
| 翻译 key 冲突          | 阻断性   | 覆盖已有翻译 |
| 4 语言缺 1 种以上      | 阻断性   | 国际化不完整 |
| Breaking change 未标注 | 非阻断性 | 记录即可     |
| JSON 格式错误          | 阻断性   | 解析失败     |
| 翻译语气不一致         | 非阻断性 | 可后续优化   |

## 类型变更的 breaking change 判定

- 删除已有字段 → 阻断性 breaking change
- 重命名已有字段 → 阻断性 breaking change
- 新增可选字段 → 非阻断性（兼容变更）
- 修改字段类型（string→number）→ 阻断性 breaking change
- 修改字段类型（`z.string()` → `z.string().email()`）→ 非阻断性（收窄）

## 翻译完成度阈值

- 4/4 语言全 → 完成
- 3/4 → 有条件完成（缺的语言后续补）
- ≤ 2/4 → 不通过，触发 re-execute
