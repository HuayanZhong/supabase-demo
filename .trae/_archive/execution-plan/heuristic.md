# Heuristic — 规划阶段最佳实践

## 扫描先行

1. 先用 Glob/Grep 扫描项目，确认是否有现成实现可复用
2. 先读目标文件的当前内容，再判断如何修改
3. 先检查项目目录结构，定位正确新建位置

## 文件分类

做计划时，将所有涉及文件分为 3 类并标注操作类型：

- **新建** — 不存在的文件，需创建
- **修改** — 已存在的文件，需编辑
- **只读** — 需要理解其内容的参考文件

## 排序原则

- 依赖文件优先：A 引用了 B，先做 B
- 底层优先：实体 / 类型 → API / 服务 → UI
- 高风险优先：破坏性变更先做，留足验证时间
- 独立文件可并行处理

## 模式复用

- 项目中已有类似实现时，优先复用其结构
- 使用标准代码分割：页面拆分组件、长逻辑拆分函数
- 框架约定优先于自定义模式

## 粒度控制

- 一个计划聚焦一个任务类型，跨领域任务拆分为子计划
- 子计划之间通过路由协调，plan 自身不处理跨域逻辑
