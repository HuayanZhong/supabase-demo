# 安全审计

**对应 Agent**：compliance-checker（`agents/compliance-checker.md`）

## 触发条件

- 任务类型匹配：quality → security
- 关键词：安全、漏洞、风险、OWASP、XSS、SQL 注入、CSRF、认证、鉴权

| --------------------------- | --------------------------------- |
| `skill/turborepo`           | 构建配置                          |
| supabase MCP                | 查看 Supabase 安全配置和 RLS 策略 |

## 执行步骤

### Step 1: 加载安全规则和检查清单

加载 OWASP Top 10 和项目相关安全检查项：

| 类别         | 检查项                               | 说明                              |
| ------------ | ------------------------------------ | --------------------------------- |
| **输入校验** | 所有用户输入是否经过 Zod schema 校验 | NestJS 的 ZodValidationPipe       |
| **认证**     | Supabase Auth 配置是否正确           | 密码策略、会话管理、MFA           |
| **鉴权**     | API 端点是否有权限控制               | Guard、Role 检查、RLS 策略        |
| **敏感信息** | .env 文件是否被 Git 忽略             | 密钥、数据库连接串、API Key       |
| **依赖漏洞** | 使用的依赖是否有已知漏洞             | pnpm audit                        |
| **XSS**      | 用户输入是否在渲染前转义             | Nuxt 默认已处理，关注 v-html      |
| **CSRF**     | 是否有 CSRF 保护                     | Nuxt 的 cookie / Supabase 的 CSRF |
| **数据库**   | RLS 策略是否合理                     | Supabase RLS 检查                 |
| **日志**     | 是否在日志中泄露敏感信息             | 密码、Token、个人数据             |

### Step 2: 逐项扫描

- **依赖漏洞扫描**：

```bash
pnpm audit
```

- **代码静态扫描**：搜索敏感信息硬编码、不安全写法
  - 搜索 `v-html` 使用
  - 搜索日志中打印 `password`、`token`、`secret` 等
  - 搜索 `.env` 中是否引用了生产环境密钥
- **Supabase 安全配置**：
  - 使用 supabase MCP 查看 RLS 策略
  - 确认 Auth 配置（密码最小长度、邮箱验证等）

### Step 3: 输出安全报告

报告格式：

```markdown
## 安全审计报告

**审计时间**：<日期>
**审计范围**：<文件列表 / 模块>

### 漏洞清单

| #   | 严重程度 | 类别     | 位置 | 描述 | 修复建议 |
| --- | -------- | -------- | ---- | ---- | -------- |
| 1   | 高危     | SQL 注入 | ...  | ...  | ...      |
| 2   | 中危     | 鉴权缺失 | ...  | ...  | ...      |

### 风险等级说明

| 等级 | 定义                               |
| ---- | ---------------------------------- |
| 高危 | 可能直接导致数据泄露或服务不可用   |
| 中危 | 有限条件下可被利用，或信息泄露风险 |
| 低危 | 违反最佳实践，但利用难度高         |

### 总结

- 高危：N 个
- 中危：N 个
- 低危：N 个
- 总体风险评级：<低/中/高>
```
