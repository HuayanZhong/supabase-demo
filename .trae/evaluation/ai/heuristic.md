# Heuristic — AI 评估阶段最佳实践

## 评估前加载顺序

```
1. workflows/ai/{task-type}.md 完成检查清单  → 评估基准
2. execution-plan/ai/（3 文件）              → 规划承诺
3. execution-engine/ai/（3 文件）            → 执行记录
```

## 评估流程

### 模型集成（integrate）

```
① API 连接验证        → 模型调用返回正确响应
② 密钥安全确认        → API key 使用环境变量，非硬编码
③ 降级方案确认        → 模型不可用时给出友好错误提示
④ 超时机制验证        → 流式/非流式都含 timeout
⑤ 多余文件检查        → 无未列入计划的文件
```

### 对话功能（chat）

```
① 消息流验证          → 流式响应按预期到达客户端
② 上下文保持验证      → 多轮对话上下文正确传递
③ 错误处理确认        → 网络/认证/限流错误捕获完善
④ 安全过滤确认        → 用户输入被适当过滤后传入模型
⑤ 多余文件检查        → 无未列入计划的文件
```

### 知识库/RAG（rag）

```
① 向量化流程验证      → embedding 到存储到检索闭环完整
② 检索质量确认        → 检索结果相关
③ 去重机制确认        → 相似文档处理
④ 上下文窗口安全      → 检索结果裁剪到模型上下文窗口内
⑤ 多余文件检查        → 无未列入计划的文件
```

### AI Agent（agent）

```
① 工具调用验证        → agent 的所有 tool 调用正确传递参数
② 循环防护确认        → 有最大调用次数限制防死循环
③ 状态管理验证        → 多步 agent 的状态在每个步骤间正确传递
④ 错误恢复确认        → 工具调用失败后 agent 能降级或给出提示
⑤ 多余文件检查        → 无未列入计划的文件
```

## 验证顺序

```
密钥安全扫描       → 无硬编码 key（阻断性）
API 连通性测试     → 模型可正常调用（阻断性）
降级方案验证       → 失败时给出友好提示（非阻断性）
超时机制验证       → timeout 设置合理（非阻断性）
```
