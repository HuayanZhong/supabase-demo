# Policy — AI 评估阶段决策策略

## 通过/不通过决策

| 评估结果                        | 结论           | 后续动作                       |
| ------------------------------- | -------------- | ------------------------------ |
| 密钥安全 + API 连通 + 降级完善  | **通过**       | 标记完成，输出报告             |
| 密钥安全 + API 连通但降级不完善 | **有条件通过** | 记录待补充降级                 |
| API key 硬编码                  | **不通过**     | 触发 re-execute                |
| 模型 API 不连通                 | **不通过**     | 触发 re-plan（换模型或修配置） |
| 流式不通                        | **不通过**     | 触发 re-execute                |

## 阻断性 vs 非阻断性判定

| 发现                         | 分类     | 理由           |
| ---------------------------- | -------- | -------------- |
| API key 硬编码               | 阻断性   | 密钥泄露       |
| 无模型调用的错误处理         | 阻断性   | 用户无反馈     |
| 流式响应中断                 | 阻断性   | 核心功能不完整 |
| RAG 检索质量偏低             | 非阻断性 | 可运行时优化   |
| 无超时机制                   | 非阻断性 | 记录待补充     |
| Agent 工具调用错误提示不友好 | 非阻断性 | 可后续优化     |

## 模型降级策略评估

| 主模型状态      | 降级要求                         |
| --------------- | -------------------------------- |
| 网络不通        | 显示友好错误提示并建议重试       |
| 限流（429）     | 自动重试 + 指数退避              |
| 持续失败（5XX） | 显示错误提示，不建议自动切换模型 |
| 返回空/无效内容 | 提示用户重试                     |

## 密钥安全评分

- 0/3 条硬编码 → 安全
- 1-2 条硬编码 → 不通过，需全部移动到 `.env`
- 密钥在代码中但已使用环境变量引用 → 通过
- 密钥文件未被 `.gitignore` 排除 → 不通过
