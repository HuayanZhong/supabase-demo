# Policy — 后端评估阶段决策策略

## 通过/不通过决策

| 评估结果                      | 结论           | 后续动作                   |
| ----------------------------- | -------------- | -------------------------- |
| 所有门禁通过 + 完整性校验通过 | **通过**       | 标记任务完成，输出评估报告 |
| 门禁通过 + lint warning       | **有条件通过** | 记录问题，标记完成         |
| Migration 缺失 down 方法      | **不通过**     | 触发 re-execute            |
| check-types/lint error        | **不通过**     | 触发 re-execute            |
| Entity 与 migration 不一致    | **不通过**     | 触发 re-plan               |
| API breaking change 未标注    | **不通过**     | 触发 re-execute（补标注）  |

## 阻断性 vs 非阻断性判定

| 发现                    | 分类     | 理由                |
| ----------------------- | -------- | ------------------- |
| Migration 无 down 方法  | 阻断性   | 无法回滚            |
| Entity 变更无 migration | 阻断性   | schema 与实际不一致 |
| API 路径变更未通知前端  | 非阻断性 | 记录移交即可        |
| 缺少 input 校验装饰器   | 非阻断性 | 可后续补充          |
| 硬编码数据库凭证        | 阻断性   | 安全隐患            |
| 模块注册遗漏            | 阻断性   | 运行时不可用        |

## 数据库变更评估策略

| 变更类型 | 评估重点                                 |
| -------- | ---------------------------------------- |
| 新增表   | migration 完整、Entity 存在、Module 注册 |
| 修改字段 | migration 含 down、数据类型安全          |
| 删除字段 | 确认该字段无前端/后端引用                |
| 新增索引 | 确认索引名规范、不影响写入性能           |
| 数据迁移 | 确认旧数据可回迁                         |

## 评估证据优先级同通用规则

- 命令输出 > 文件内容 > git diff > grep/glob > AI 自主判断
