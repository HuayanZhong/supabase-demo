# 国际化变更 Workflow

## 触发条件

- 任务类型为 `i18n`（关键词：翻译、多语言、Locale）
- 执行 Agent：`frontend-architect`
- 覆盖路径：`apps/frontend/`（翻译文件本身由 shared 路由处理）

## 准备工作

### 1. 加载资源

| 资源                         | 说明                                              |
| ---------------------------- | ------------------------------------------------- |
| `rules/frontend/i18n.md`     | 国际化规范（locale code、翻译文件管理、切换方式） |
| `rules/frontend/comments.md` | 注释规范                                          |
| `skill/nuxt-ui`              | Nuxt UI 组件知识                                  |
| `execution-plan/frontend/`   | 规划指引：约束/最佳实践/决策策略                  |

### 2. 确认变更范围

- **新增 key**：哪些文本需要翻译
- **修改 key**：现有 key 的文案变更
- **删除 key**：废弃的翻译条目
- **新增语言**：需要添加哪种 locale

## 执行步骤

### Step 1 — 确认翻译范围

- 搜索前端代码中所有使用 `t('...')` 或 `$t('...')` 的位置
- 确认哪些文本需要新增/修改/删除翻译 key
- 区分：
  - **翻译文件变更**（增删改 key/语言）→ 转 **shared** 路由，本 workflow 仅记录
  - **代码中引用变更** → 本 workflow 执行

### Step 2 — 代码关联（前端侧）

- 在 Vue 模板中使用 `$t('module.key')` 引用翻译
- 在 `<script setup>` 中使用 `useI18n()` 的 `t` 函数
- 确认 key 路径与翻译文件中的层级一致
- 如涉及 Nuxt UI 组件的 label/text prop，使用 `:label="$t('key')"` 绑定

### Step 3 — 切换验证

- 确认所有 4 语言（zh-CN / en / ja / ko）的翻译已同步更新
- 切换语言后页面文本正确渲染
- 确认 `ULocaleSelect` 选中状态正确
- 确认不存在未翻译的硬编码文本

### Step 4 — 清理

- 移除不再使用的翻译 key（先确认代码中无引用）
- 统一 key 命名风格：按功能模块分组（`auth.login`、`common.submit`）

## 完成检查

- [ ] 所有用户可见文本已通过 `$t()` 或 `t()` 引用
- [ ] 翻译 key 按功能模块分组、命名一致
- [ ] 不存在未翻译的硬编码文本
- [ ] 切换日语/韩语等非拉丁语言后布局不溢出
- [ ] 4 语言翻译已同步更新（如有变更，转 shared 路由处理翻译文件）
- [ ] lint 通过（`pnpm lint`）
- [ ] format 通过（`pnpm format`）

## 输出

- 变更摘要：修改/新增/删除了哪些翻译 key
- 翻译文件变更清单（转 shared 路由执行）
- 代码变更清单：更新了哪些组件/页面的 `$t()` 调用
