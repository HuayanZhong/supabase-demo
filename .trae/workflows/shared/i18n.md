# 翻译文件变更 Workflow

## 触发条件

- 任务类型为 `i18n`（关键词：翻译、Locale、新增语言）
- 执行 Agent：`frontend-architect`
- 覆盖路径：`packages/i18n/`（翻译文件本身）、`apps/frontend/nuxt.config.ts`（locale 配置）

## 准备工作

### 1. 加载资源

| 资源                           | 说明                                                      |
| ------------------------------ | --------------------------------------------------------- |
| `rules/frontend/i18n.md`       | 国际化规范（locale code、翻译文件管理、新增语言检查清单） |
| `rules/frontend/comments.md`   | 注释规范                                                  |
| `apps/frontend/nuxt.config.ts` | Nuxt i18n 配置                                            |
| `execution-plan/shared/`       | 规划指引：约束/最佳实践/决策策略                          |

### 2. 确认变更范围

- **新增 key**：哪些功能模块需要新增翻译条目
- **修改文本**：现有 key 的文案内容变更
- **删除 key**：废弃的翻译条目（需先确认代码中无引用）
- **新增语言**：需要添加哪种 locale（确认 BCP 47 格式）
- **仅代码中引用变更**（无翻译文件变化）→ 不进入本 workflow，走 frontend 路由

## 执行步骤

### Step 1 — 确认翻译变更范围

- 读取 `packages/i18n/locales/` 下现有翻译文件，了解当前语言覆盖情况
- 读取 `packages/i18n/src/index.ts`，了解现有 export 结构
- 明确本次变更是新增 key、修改文本、删除 key 还是新增语言
- 如是新增语言，先确认 `@nuxt/ui/locale` 中是否存在该语言的导出

### Step 2 — 修改翻译文件

- 修改 `packages/i18n/locales/` 下对应的语言文件
- **必须同步更新所有 4 语言**（`zh-CN.json`、`en.json`、`ja.json`、`ko.json`），保持 key 结构一致
- 新增 key 按功能模块分组（如 `common.submit`、`auth.login`），遵循现有嵌套结构
- 删除 key 时确认所有语言文件中均已移除
- 新增语言时创建 `<code>.json` 文件，按现有文件结构填充

### Step 3 — 更新 packages/i18n 导出配置

- 在 `packages/i18n/src/index.ts` 中 import 并 export 新增语言文件
- 更新类型定义，确保新增语言纳入类型系统
- 在 `packages/i18n/package.json` 的 `exports` 中添加新增语言的路径映射

### Step 4 — 同步 nuxt.config.ts

- 在 `apps/frontend/nuxt.config.ts` 的 `i18n.locales` 数组中添加/删除对应 locale
- 每个 locale 条目包含 `{ code, name, file }` 三字段
- 新增语言时同步确认 `@nuxt/ui/locale` 的导入和 `UApp` 的 locale prop 绑定

### Step 5 — 验证

- 确认 4 语言翻译文件已同步更新，无遗漏 key
- 运行 `pnpm check-types` 确认类型检查通过
- 运行 `pnpm build` 确认编译通过
- 启动前端 dev server，切换各语言验证页面文本正确渲染
- 确认 `ULocaleSelect` 选中状态正确
- 确认新增语言后布局不溢出

## 完成检查

- [ ] 4 语言翻译文件（zh-CN / en / ja / ko）同步更新，key 结构一致
- [ ] 翻译 key 按功能模块分组、命名一致
- [ ] `packages/i18n/src/index.ts` export 已更新
- [ ] `packages/i18n/package.json` exports 已更新
- [ ] `apps/frontend/nuxt.config.ts` locale 配置已同步
- [ ] 新增语言时 `@nuxt/ui/locale` 存在对应语言导出
- [ ] `pnpm check-types` 通过
- [ ] `pnpm build` 通过
- [ ] 切换各语言后 UI 正常，无硬编码文本遗漏

## 输出

- 变更摘要：新增/修改/删除了哪些翻译 key，或新增了哪种语言
- 文件变更清单：修改了 `packages/i18n/` 和 `apps/frontend/nuxt.config.ts` 的哪些文件
- 验证结果：typecheck、build 和多语言切换验证结果
