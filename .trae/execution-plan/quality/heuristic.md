# Heuristic — 质量规划最佳实践

## 测试策略

| 测试类型 | 覆盖范围               |
| -------- | ---------------------- |
| 单元测试 | 工具函数、Service 方法 |
| 集成测试 | API 端点、数据库交互   |
| E2E 测试 | 关键用户流程           |
| API 测试 | 所有公开端点           |

## 审查流程

1. 先读业务背景，理解代码意图
2. 检查代码结构是否符合该领域的规范
3. 检查是否有明显的边界情况遗漏
4. 检查异常处理是否完备
5. 给出可执行的具体修改建议

## 性能分析流程

1. 先确认问题表现（加载慢 / 操作卡顿 / 内存高）
2. 选合适工具收集数据（前端用 Lighthouse，后端用 Clinic.js）
3. 定位瓶颈（渲染 / 网络 / 数据库 / 计算）
4. 提出优化方案并估算收益

## 安全检查清单

- [ ] 依赖中是否有已知漏洞（`npm audit`）
- [ ] 是否有硬编码密钥或敏感信息
- [ ] 用户输入是否经过校验
- [ ] API 端点是否有权限控制
- [ ] 是否有 SQL 注入或 XSS 风险
- [ ] 环境变量是否有合理的默认值

## 报告规范

- 问题标注严重级别：阻断性 / 主要 / 次要 / 建议
- 每个问题包含：位置（文件+行号）→ 问题描述 → 修复建议
- 安全类问题：追加 CVE 编号和受影响版本
