# Policy — 共享包规划决策策略

## 类型放置决策

| 场景               | 策略                                 |
| ------------------ | ------------------------------------ |
| 单个组件用到的类型 | 组件文件内定义，不放入共享包         |
| 单个页面用到的类型 | 页面所在目录定义，不放入共享包       |
| 两个以上模块共用   | 放入 `packages/types/`               |
| API 请求/响应结构  | 放入 `packages/types/`（前后端共享） |

## Schema 变更决策

| 场景             | 策略                                 |
| ---------------- | ------------------------------------ |
| 新增字段         | 非必填，不影响现有代码               |
| 删除字段         | 检查所有引用方，确认无线索引用       |
| 改字段类型       | 标注为破坏性变更，同步更新所有引用方 |
| 合并/拆分 Schema | 保留旧引用别名，逐步迁移             |

## i18n 变更决策

| 场景        | 策略                               |
| ----------- | ---------------------------------- |
| 新增 key    | 同步更新 4 个语言文件              |
| 修改 key 值 | 检查文案变更是否影响其他引用方     |
| 删除 key    | 搜索代码确认无引用后再删除         |
| 新增语言    | 更新 `packages/i18n/` 每一层的配置 |

## 冲突处理

| 场景                           | 策略                           |
| ------------------------------ | ------------------------------ |
| 前端和后端对同一类型有不同需求 | 以共享包中定义为基准，两方适配 |
| 新增类型与现有类型重叠         | 优先复用/扩展现有类型          |
| 包间循环依赖                   | 提取公共部分到更底层的包       |
