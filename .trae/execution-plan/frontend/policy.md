# Policy — 前端规划决策策略

## 组件设计决策

| 场景                | 策略                                          |
| ------------------- | --------------------------------------------- |
| 组件逻辑复杂        | 拆分为展示组件 + 容器组件                     |
| 多处页面用到相同 UI | 提升到 `components/common/`                   |
| 需要用户输入        | 优先使用 Nuxt UI 表单组件（UForm + UInput）   |
| 需要列表/表格       | UTable（轻量） > 自定义 Table（需复杂交互时） |
| 需要弹窗/抽屉       | UModal / UPopover > 自建 overlay              |

## 数据获取决策

| 场景                 | 策略                                      |
| -------------------- | ----------------------------------------- |
| 单页面用到的数据     | 直接在页面中用 composable                 |
| 多页面共享的数据     | composable 放在 `composables/` 下复用     |
| 服务端渲染需要的数据 | 使用 Nuxt `useAsyncData` / `useFetch`     |
| 需要实时更新的数据   | 标注"需确认后端是否有 WebSocket/SSE 支持" |

## 布局决策

| 场景            | 策略                                       |
| --------------- | ------------------------------------------ |
| 仪表盘类页面    | 使用 `UDashboardGroup` + `UDashboardPanel` |
| 列表页 + 详情页 | 列表页用 UTable，详情页用 UCard 布局       |
| 登录/注册       | 居中卡片布局，不搞分栏/大面积纯色背景      |
| 导航结构        | `UNavigationMenu` > 自建导航               |

## 状态决策

| 场景           | 策略                                   |
| -------------- | -------------------------------------- |
| 组件内状态     | `useState` 或 `ref`                    |
| 页面间共享状态 | composable + `useState`（SSR 安全）    |
| 全局登录态     | 复用已有的 `useAuth` composable        |
| 表单状态       | UForm 自带表单验证，不额外引入 form 库 |

## 交付决策

| 场景         | 策略                                      |
| ------------ | ----------------------------------------- |
| 新页面完成   | 列出新增的翻译 key，标注"需路由到 shared" |
| 新组件完成   | 标注被哪些页面引用                        |
| 组件修改完成 | 标注影响到的页面和可能回归范围            |
