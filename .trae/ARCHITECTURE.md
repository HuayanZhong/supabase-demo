# Trae AI — 治理架构 v3

## 从 v2 的 3 层到 v3 的 5 层

v2 解决了"编码质量"（pre-commit 门禁），但"任务收尾"和"经验进化"仍是死角。
v3 补上这两个断点：**把软约定变成 alwaysApply 里的硬检查**。

```
用户任务
    │
    ▼
┌──────────────────────────────────────────────────────┐
│  ① alwaysApply 注入                                 │
│     语言 + 交互 + 安全 + 预检 + 收尾检查清单          │
│     ai-safety.md — IDE 每次对话自动加载              │
└──────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────┐
│  ② 领域规则（场景触发）                               │
│     rules/{domain}/*.md + code-style / comments 等    │
│     按任务关键词自动匹配加载                          │
└──────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────┐
│  ③ Pre-commit 硬门禁                                 │
│     oxfmt → oxlint → check-types                     │
│     走不过去 = 不能提交                               │
└──────────────────────────────────────────────────────┘
    │
    ▼ 任务编码完成
┌──────────────────────────────────────────────────────┐
│  ④ Post-task 收尾闭环  【v3 新增】                    │
│     追踪摘要  →  对话中输出（用户 + agent 可见）      │
│     经验写入  →  memory/experience/{task-id}.json     │
│     使用率记  →  同一文件内嵌 stats 字段              │
│     规则自检  →  覆盖率 / 完成率 / 使用率             │
└──────────────────────────────────┬───────────────────┘
                                   │ 数据流入
                                   ▼
┌──────────────────────────────────────────────────────┐
│  ⑤ Evolution 自迭代  【v3 激活】                      │
│     经验累积达 N 条（默认 5）→ 自动提议聚合          │
│     分析 → 提案 → 人工确认 → 应用 → 验证              │
│     每次 evolution 只改 1 个治理文件                  │
└──────────────────────────────────────────────────────┘
```

## 强制力分布

| 层级          | 机制             | 强制力             | 验证方式                   |
| ------------- | ---------------- | ------------------ | -------------------------- |
| ① alwaysApply | IDE 每次注入     | 高                 | agent 必须遵循检查清单     |
| ② 领域规则    | 场景关键词匹配   | 中                 | 追踪摘要中报告             |
| ③ Pre-commit  | hook 硬跑        | **硬（不可绕过）** | CI 失败即阻挡              |
| ④ Post-task   | alwaysApply 清单 | 高                 | experience/ 有文件即有产出 |
| ⑤ Evolution   | 数据驱动触发     | 中                 | experience/ 文件数 ≥ 阈值  |

## 关键改进

| v2 痛点                   | v3 修复                                   |
| ------------------------- | ----------------------------------------- |
| 记忆经验写入从未触发      | 写入 alwaysApply 收尾清单，任务不完不结束 |
| 元进化循环不启动          | experience/ 累积到 5 条自动提议聚合       |
| 日志 180 行过度设计       | 砍到 48 行：只保留追踪摘要 + 经验数据格式 |
| AGENTS.md 引用 7 层旧模型 | 修正为 5 层 v3 模型                       |
| 领域规则 agent 不可见     | 收尾清单强制自检使用率                    |

## 设计原则

1. **只信硬约束** — 文档约定靠不住，流程清单和 hook 才靠得住
2. **砍掉死肉** — 不用的格式定义、不触发的循环直接归档
3. **数据驱动** — evolution 靠 experience/ 文件数触发，不靠"记得"
4. **用户可见** — 追踪摘要给用户看，经验文件给 agent 看，各取所需
