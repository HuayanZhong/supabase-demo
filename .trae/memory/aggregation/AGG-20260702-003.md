## 发现归纳

12 个失败可归纳为 6 类缺失机制：

| 类别            | 覆盖用例       | 缺失原因                             |
| --------------- | -------------- | ------------------------------------ |
| Checkpoint 系统 | U1、V1、X1     | 无标准化的任务状态持久化             |
| Scope 变更      | U2、U3、W3、W4 | scope 仅任务开始时设定，不可中途变更 |
| 部分提交        | U4             | 依赖链要么全提交要么全回滚           |
| 频繁变更检测    | U5             | 无方向变更计数器                     |
| 运行时 MCP 降级 | V3             | 降级方案仅覆盖初始可用性检查         |
| 约束豁免        | W2             | 用户无法跳过评估约束                 |

## 矛盾检测

| 新机制                | 现有约束       | 是否矛盾                                        |
| --------------------- | -------------- | ----------------------------------------------- |
| 约束豁免（override）  | GUARD 必须拦截 | 无矛盾——override 要求用户主动 \                 |
| 运行时降级（degrade） | 降级方案库     | 无矛盾——同一套方案库，触发点扩展                |
| Scope 变更            | scope 守卫     | 无矛盾——变更后更新 scope 声明                   |
| Checkpoint            | git snapshot   | 无矛盾——checkpoint 是语义级，snapshot 是 git 级 |
