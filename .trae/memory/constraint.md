# Constraint — 记忆系统硬约束

## 安全约束

- 记忆数据不得包含敏感信息：API Key、密码、令牌、个人隐私
- 各子目录通过 `.gitignore` 排除，确保记忆数据不提交到版本控制
- Trae 内置记忆（全局/项目）仅写入摘要，完整数据始终存储在 `.trae/memory/` 下

## 数据完整性

- 每条记录必须有时间戳（`ISO 8601` 格式，如 `2026-07-01T10:30:00Z`）
- 每条记录必须有来源标识（`source` 字段，标记产生该记录的任务/会话）
- 已写入的记忆数据不得原地修改，采用追加模式（新版本覆盖标记，旧版本保留在历史中）
- 删除操作只能标记为 `deprecated`，不物理删除

## 读写约束

| 操作             | 约束                                           |
| ---------------- | ---------------------------------------------- |
| 写入 profile/    | 必须有证据支撑，不得凭猜测写入用户偏好         |
| 写入 experience/ | evolution 聚合完成后才能写入                   |
| 写入 patterns/   | 同一模式出现 ≥ 3 次才能写入                    |
| 写入 sessions/   | 会话结束时统一归档，中间状态不持久化           |
| 跨会话读取       | 必须先校验数据的时效性（超期数据标记为 stale） |

## 容量约束

- `sessions/` 保留最近 30 天的会话记录
- `experience/` 保留最近 60 天的经验快照
- `patterns/` 和 `profile/` 永久保留（可手动清理）
- 超出保留期的数据压缩为摘要后删除原始文件
