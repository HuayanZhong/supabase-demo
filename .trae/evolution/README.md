# Evolution — 元治理 v2

## 职责

经验驱动的治理体系自迭代。从完成任务中收集数据，识别模式，提出改进。

```
任务完成 → 收集经验数据 → 聚合分析 → 提案 → 人工确认 → 应用
```

## 文件结构

```
.trae/evolution/
├── README.md         # 本文档
├── constraint.md     # 硬约束（不可违反的红线）
├── heuristic.md      # 进化流程（收集 → 提案 → 验证）
└── policy.md         # 决策策略
```

## 进化周期

| 阶段 | 触发条件         | 产出                       |
| ---- | ---------------- | -------------------------- |
| 收集 | 每次任务完成后   | 经验记录                   |
| 聚合 | 累计 N 条 / 定期 | 聚合报告                   |
| 分析 | 聚合完成后       | 根因 + 模式 + 规则变更检测 |
| 提案 | 分析完成后       | 改进方案（待人工确认）     |
| 应用 | 提案确认后       | 修改治理文件               |
| 验证 | 应用后           | 观察后续效果               |

## 核心原则

- **基于证据** — 至少 1 条真实经验，不得凭空建议
- **安全不可降级** — ai-safety.md 的安全约束不可被推翻
- **不自改** — evolution 自身变更需人工确认
- **可回滚** — 每次变更备份至 `memory/backups/`
