# Heuristic — 进化流程 v2

## 概述

进化周期：收集经验 → 聚合分析 → 提案 → 应用 → 验证。
所有提案类变更需人工确认后方可执行。

```
① 收集 ← 每次任务完成后写入经验数据
    ↓
② 聚合 ← 累计 N 次或定期触发
    ↓
③ 分析 ← 根因 + 模式识别 + 规则变更检测
    ↓
④ 提案 ← 生成改进方案
    ↓
⑤ 应用 ← 人工确认后执行
    ↓
⑥ 验证 ← 观察后续任务效果
```

## 进化日志

```
[EVOLVE:collect]   OK  | 收集经验数据
[EVOLVE:aggregate] OK  | 聚合完成
[EVOLVE:analyze]   OK  | 根因分析完成
[EVOLVE:propose]   OK  | 生成提案（待确认）
[EVOLVE:apply]     OK  | 应用变更（需人工确认）
[EVOLVE:verify]    OK  | 验证完成
```

## 规则变更检测（③ 分析阶段）

当 rules/ 目录有变更时：

1. 读取 `rules/README.md` 获取当前规则清单
2. 对比上次基线，检测新增/删除/修改的文件
3. 分析变更对编码实践的潜在影响

高风险规则（安全/凭证/部署）标记为 `[EVOLVE:rule] CRITICAL`

## 提案约束

- 提案必须基于经验数据，不得凭空建议
- 修改 `ai-safety.md` 必须人工确认
- 应用变更后至少观察 3 次任务再决定保留或回滚
