# Step 2: 协作效率分析

分析用户与 AI 之间的协作效率，评估意图理解和任务执行质量。

## 执行

### 2.1 意图理解

逐轮回看用户消息，评估以下指标：

| 指标             | 评估方法                                             | 判定标准                           |
| ---------------- | ---------------------------------------------------- | ---------------------------------- |
| **一次理解率**   | 用户提出需求后，AI 是否第一次就给出正确响应          | 用户无纠正 / 无需后续补充说明 = ✅ |
| **纠正次数**     | 用户明确指正 AI 理解偏差的次数                       | 计数累加                           |
| **平均纠正深度** | 纠正所需的额外对话轮次                               | 1 轮 = 浅，2-3 轮 = 中，>3 轮 = 深 |
| **模糊需求处理** | 用户需求不明确时，AI 是否主动澄清（AskUserQuestion） | 主动澄清 = ✅，猜测执行并出错 = ❌ |

**输出**：一次理解率 = `$first_try_correct / $total_tasks`，列出所有需要纠正的场景和原因。

### 2.2 任务执行效率

| 指标               | 评估方法                                                             |
| ------------------ | -------------------------------------------------------------------- |
| **重复工作检测**   | 是否存在相同或类似的工作被执行多次（如重复执行同一审计）             |
| **工具选择合理性** | AI 是否选择了正确的工具完成任务（Read vs Write，RunCommand vs 工具） |
| **耗时偏差**       | 对比任务日志中的"预估"和"实际"时间，识别偏差 > 50% 的任务            |
| **任务碎片化**     | 一个需求是否被拆成过多小步骤、有无不必要的子智能体调用               |

**输出**：识别出的效率问题清单，每条附发生位置和影响。

### 2.3 无效工作统计

统计以下类型的无效工作：

- 重复执行的任务
- 用户要求停止或撤回的操作
- 生成后未被使用的代码
- 误解需求导致的重做

**输出**：无效工作次数 `$wasted_work_count`，估算影响（耗时分钟数）。

### 2.4 专家系统调度评估

评估本次会话中 Subagent 和 Skill 的调度效果：

| 指标                | 计算方法                     | 判定标准                                       |
| ------------------- | ---------------------------- | ---------------------------------------------- |
| **Subagent 命中率** | Subagent 调用次数 / 总轮次   | ≥ 20% 合理；0% 但存在并行任务场景 → 调度缺失   |
| **Skill 命中率**    | Skill 调用次数 / 总轮次      | ≥ 15% 合理；0% 但有多类专家知识需求 → 调度缺失 |
| **应调未调场景**    | 对照 Skill/Subagent 场景匹配 | 列出未调用的 Skill/Subagent 及其触发条件       |

### 2.5 协作模式总结

归纳用户与 AI 之间的协作风格：

- 用户偏好：先给方向 vs 给完整需求
- AI 适应性：是否从前期错误中学习并调整
- 沟通效率：是否需要频繁澄清

## 输出摘要

```
[{步骤2 协作}] 一次理解率 {N}/{N} | 纠正 {N} 次 | 无效工作 {N} 次 | 耗时偏差 {N}% | Subagent 命中率 {P}% | Skill 命中率 {P}%
```
