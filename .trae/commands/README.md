# Trae 命令

`.trae/commands/` 目录下的命令通过 `/` 前缀在对话中直接触发。

## 可用命令

| 命令     | 文件       | 用途                                                    |
| -------- | ---------- | ------------------------------------------------------- |
| `/audit` | `audit.md` | 全量扫描治理架构，检查 Skills/MCP/Hooks/规则/配置一致性 |
| `/retro` | `retro.md` | 复盘 AI 与用户协作过程，识别短板输出改进建议            |

> 各命令的详细步骤存放在对应的 `命令名/` 子目录下，编排器按步骤逐一加载。

## 使用方式

在对话中输入 `/命令名` 即可触发 AI 执行对应流程。

## 命令清单

### `/audit`

全量扫描项目治理架构，覆盖 **资源存在性 + 交叉引用一致性 + 语义检查** 三个维度：

**存在性维度**（单个资源是否正常存在）：

- Skills junction 完整性（`.agents/skills/` ↔ `.trae/skills/`）
- MCP 工具描述文件存在性
- 规则文件 frontmatter 元数据完整性
- Hooks 绑定完整性（`hooks.json` ↔ `.ps1` 文件 ↔ 内部规则引用）
- 环境变量与 `.env.example` 一致性

**交叉引用维度**（资源间的引用关系是否正确）：

- `AGENTS.md` ↔ 规则文件（树形图、规则表、生效方式、适用场景）
- `AGENTS.md` ↔ `hooks.json`（生命周期图一致性）
- `AGENTS.md` ↔ `hooks/README.md`（生命周期图、规则映射一致性）
- `hooks/README.md` ↔ `hooks.json`（事件明细表一致性）
- `hooks/README.md` ↔ `.ps1` 脚本（行为描述一致性）
- `rules/README.md` ↔ `AGENTS.md` / `hooks.json`（三路一致性）
- `commands/README.md` ↔ `audit.md`（命令描述覆盖度）
- 规则文件间的交叉引用有效性

执行后输出结构化的审计报告。

### `/retro`

复盘 AI 与用户的协作过程，覆盖 **意图理解、执行效率、治理依从性、错误模式** 四个维度：

**分析数据源**：

- 对话消息（含已压缩的上下文摘要）
- 任务日志（本会话已有的结构化日志）
- hooks 上下文注入记录
- 历史复盘报告（`.trae/snapshots/retro/`）

**产出物**：

- 协作效率评估（一次理解率、纠正次数、无效工作）
- 治理健康状况（hook 触发率、规则引用率、日志完整度）
- 问题与改进建议（按高/中/低优排序）
- 报告存档至 `.trae/snapshots/retro/`，支持趋势对比

> 本命令为只读操作，不修改任何项目文件。
