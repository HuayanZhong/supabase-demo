# Step 3: 规则审计

> **Skill 注入**：开始本步骤前，先调用 `Skill("TRAE-code-review")` 获取审计方法论框架（范围确定+上下文收集），然后按以下逻辑执行。

检查规则文件存在性、引用有效性及 AGENTS.md 一致性。

## 3a. 存在性检查

- 递归扫描 `.trae/rules/` 下所有 `.md` 文件，记录相对路径列表
- 排除 `README.md` 后，对每个规则文件：
  - 读取前 5 行，检查是否以 `---` 开始和结束（YAML frontmatter）
  - 检查 frontmatter 中是否包含 `alwaysApply` 字段（值必须为 `true` 或 `false`）
  - 检查 frontmatter 中是否包含 `description` 字段（值不能为空）
  - `README.md`（目录概览文件）豁免以上所有检查

## 3b. 文件间引用有效性

- 对每个 `.md` 文件，扫描内容中匹配正则 `\.trae/rules/[a-zA-Z0-9_\-/]+\.md` 的路径
  - 验证目标文件是否存在，不存在则报告"引用目标不存在"
- 扫描形如 `[filename.md（xxxx）]` 或 `filename.md（xxxx章节）` 的锚点引用
  - 读取目标文件，验证存在对应的 `## xxxx` 级别标题，不存在则报告"锚点不存在"

## 3c. AGENTS.md ↔ 规则文件 一致性

- **树形图验证**：读取 `AGENTS.md` 中"治理架构"代码块，与实际目录结构逐行对比
  - 忽略缩进空格差异，提取文件名/目录名，检查是否存在
  - 缺失分支 → 追加（自动修复）；多余分支 → 输出待移除列表，等待用户确认
- **规则表验证**：读取 `AGENTS.md` 中"规则体系"Markdown 表格，与实际文件列表比对
  - 未收录的规则文件 → 追加到规则表（自动修复）
  - 已收录但文件已删除的 → 从规则表中移除（输出列表，等待用户确认）
- **深度一致性检查**（语义匹配）：
  - 生效方式 `alwaysApply` 含义一致：`true` = 始终生效，`false` = 智能生效
  - 适用场景与 `description` **语义一致**（描述同一件事即可，不要求逐字匹配）
  - 规则表行数 = 规则文件数（不含 README.md），不匹配时报告差异明细

## 输出摘要

```
[{步骤3 规则}] 总 {N} 个 | frontmatter缺失 {N} 个 | 引用异常 {N} 处 | 锚点异常 {N} 处 | AGENTS.md 不一致 {N} 处
```
