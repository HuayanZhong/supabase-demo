# Step 7: 修复验证

> **Skill 注入**：开始本步骤前，先调用 `Skill("TRAE-code-review")` 获取审计方法论框架（范围确定+上下文收集），然后按以下逻辑执行。

对自动修复项进行再检查，确保修复生效。

## 执行

- 新建的 junction → 重新读取 `.trae/skills/` 确认已创建
- 更新的 README → 重新读取确认内容已更新
- 追加的 AGENTS.md 规则表行 → 重新读取确认已追加
- 追加的 `.env.example` 变量 → 重新读取确认已追加
- 重新验证失败的修复项，标记为"需人工介入"
