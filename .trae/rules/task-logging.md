## 强制要求

每次任务完成后，**必须在对话末尾**输出以下格式的结构化日志。不得跳过此步骤。

## 输出格式

```markdown
## 任务日志

领域: {backend/frontend/shared/quality}
任务: {一句话描述}

### 资源覆盖率

- 规则命中: {触发的规则数}/{总规则数} 条 ({百分比}%)
- Skills 调用: {实际调用的}/{已安装的} 个 ({百分比}%)
- MCP 使用: {实际调用的}/{已注册的} 个 ({百分比}%)

### 调用链路

- 规则: {触发规则列表}
- Skills: {实际调用的技能包}
- MCP: {实际调用的 MCP 工具}
- 子智能体: {实际使用的子智能体类型}

### 变更清单

- 创建: {新建文件路径列表}
- 修改: {修改文件路径列表}
- 删除: {删除文件路径列表}

### 验证

- lint: ✅ / ❌
- format: ✅ / ❌
- typecheck: ✅ / ❌

### 耗时

- 预估: {时间}
- 实际: {时间}
```

## 资源覆盖率统计方法

| 指标        | 分子                                  | 分母                              |
| ----------- | ------------------------------------- | --------------------------------- |
| 规则命中    | 本次任务中实际触发/读取过的规则文件数 | `.trae/rules/` 下总规则数         |
| Skills 调用 | 本次任务中实际调用过的技能包数        | `.trae/skills/` 下总技能包数      |
| MCP 使用    | 本次任务中实际调用过的 MCP 工具数     | `.trae/mcp.json` 中注册的总工具数 |
