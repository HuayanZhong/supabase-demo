# 安全约束

## 异常处理与回滚

- 文件修改前：将待修改文件的当前内容以快照形式持久化到 `.trae/snapshots/` 目录
  - 快照文件命名格式：`{timestamp}-{相对路径}.snapshot`
  - 同一任务的所有快照归入以时间戳命名的子目录
- 修改失败：子智能体返回错误详情 → 主智能体评估能否修复 → 无法修复则从快照回滚
- 严重错误（编译不通过、lint 失败）：不得提交为最终结果
- 如用户不满意当前结果，主智能体应回退到上一个用户确认的状态

## 安全边界

- API Key、数据库密码必须通过 `@supabase/config` 的环境变量读取
- 禁止在源码中硬编码任何密钥
- 涉及删除文件、表结构变更、数据销毁的操作，必须先向用户确认
- **涉及文件合并、拆分、重命名、目录结构调整等破坏性结构变更，必须先提出方案经用户同意后执行**

## MCP 工具使用约束

| 工具类型                              | 允许场景                                   | 禁止场景                                 | 前置要求                            |
| ------------------------------------- | ------------------------------------------ | ---------------------------------------- | ----------------------------------- |
| `tavily_search`                       | 用户要求的网络搜索/网页内容获取            | 搜索未经授权的内网地址                   | 需用户明确指定搜索内容              |
| `search-components` / `get-component` | 用户要求的 Nuxt UI 组件查询                | —                                        | 仅用于文档查询，不修改文件          |
| `execute_command`                     | 运行用户明确要求的命令（git、pnpm、turbo） | 扫描系统目录、读取敏感文件、修改系统配置 | 命令内容必须向用户展示确认          |
| `ssh_execute`                         | 用户明确要求的远程操作                     | 连接未经用户授权的服务器                 | 连接目标必须经用户确认              |
| `upload_file`                         | 前端测试中的文件选择操作                   | 读取系统密钥文件、私密文档               | 文件路径需在项目目录内              |
| `execute_sql`                         | 用户要求的数据库查询/迁移                  | DROP TABLE/TRUNCATE 等破坏性操作         | 执行前必须展示 SQL 内容并获用户确认 |
| `evaluate_script`                     | 前端调试/测试中的 JS 评估                  | 注入恶意脚本、窃取页面数据               | 仅用于自动化测试场景                |
| `apply_migration`                     | 用户要求的数据库迁移                       | 无用户确认的自动迁移                     | 必须展示迁移文件内容后执行          |
