## 测试文件结构

- 前端测试文件放在 `apps/frontend/test/{unit|e2e|nuxt}/` 下
- 测试文件命名：`*.test.ts` 或 `*.spec.ts`
- 单元测试使用 Vitest + `@vue/test-utils`
- E2E 测试使用 Playwright，优先通过 `webapp-testing` Skill 辅助
- 后端测试需要时才创建，使用 NestJS 的 `@nestjs/testing` TestFactory

## 必须编写测试的场景

以下场景必须编写对应的单元测试或集成测试：

| 场景                       | 必须测试的内容                                 | 测试类型 |
| -------------------------- | ---------------------------------------------- | -------- |
| 新增/修改 Service 核心逻辑 | 核心业务路径 + 边界条件 + 异常分支             | 单元测试 |
| 新增 DTO/输入校验          | 校验规则的正确性（合法输入通过、非法输入拒绝） | 单元测试 |
| 涉及数据变更的 API         | API 的完整请求-响应链路（含数据库操作）        | 集成测试 |
| 安全敏感接口（认证/授权）  | 未认证/未授权访问的正确拒绝                    | 集成测试 |
| Bug fix                    | 再现 Bug 的测试用例 + 修复后验证               | 单元测试 |
| 复杂查询/聚合逻辑          | 查询结果的正确性 + N+1 预防                    | 集成测试 |

## 不需要编写测试的场景

- 单纯的 DTO/Entity 类定义（不含自定义逻辑）
- 配置类、常量定义、枚举定义
- Controller 路由定义（路由转发逻辑由集成测试覆盖）
