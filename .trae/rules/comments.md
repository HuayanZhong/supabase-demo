> **MANDATORY** — 新建文件必须包含至少 1 条注释（文件级说明或关键函数行内注释）。

## 核心要求

- 不需要为每个函数/变量都写注释，只在代码意图不明显时加
- 注释解释"为什么这么做"，而非"做了什么"（代码本身应能看出做了什么）
- JSDoc 只用于公开 API（export 的函数/类/类型），内部函数仅在逻辑复杂时使用
- 行内注释使用 `//`，和代码之间空一格
- 块注释 `/* */` 仅用于许可证 header 或文件级别说明，不使用块注释禁用代码块
- TODO 注释格式：`// TODO(责任人): 原因`
- FIXME 标记：`// FIXME: 需要修复的问题`
- HACK 标记：`// HACK: 临时解决方案，待后续优化`
- 不写 @author、@date 等冗余标签

## 修改文件时的要求

- 修改已有文件时优先使用 SearchReplace 做局部编辑
- 必须用 Write 全量重写时，需确保已确认保留原文件的所有注释内容
- **删除注释比漏加注释更危险**——不确定是否冗余的先保留

## 常见违规

| ❌ 错误的注释                                             | 问题                | ✅ 正确处理              |
| --------------------------------------------------------- | ------------------- | ------------------------ |
| `// 获取当前用户信息`（next to `getSession()`）           | 代码自说明          | 删除                     |
| `// 登录表单字段`（next to `const form = reactive(...)`） | 代码自说明          | 删除                     |
| `/** 退出登录 */` on `handleLogout()`                     | 内部函数不该用JSDoc | 改为 `//` 或删除         |
| `// TODO: 修复xx`                                         | 缺少责任人          | `// TODO(owner): 修复xx` |
| 新文件完全没有注释                                        | 违反 MANDATORY      | 添加文件级说明注释       |
