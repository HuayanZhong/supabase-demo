# 注释约束

## 核心原则

- 注释解释**为什么**（Why），不是**是什么**（What）
- 代码本身能清晰表达的，不写注释
- 所有注释必须使用中文

## 格式

| 上下文                  | 格式                | 说明                |
| ----------------------- | ------------------- | ------------------- |
| TypeScript / JavaScript | `// 注释内容`       | 行内注释            |
| 导出函数 / 公开 API     | `/** 注释内容 */`   | JSDoc，仅限复杂业务 |
| Vue `<template>`        | `<!-- 注释内容 -->` | 仅作区块分隔        |
| Vue `<script>`          | `// 注释内容`       | 同 TS 规则          |

## 必须注释的场景

- **复杂业务逻辑**：规则、边界条件、Hack
- **魔数/非常见常量**：值来源或含义
- **非预期行为**：为什么这么写而不那么写
- **TODO / HACK / FIXME / WORKAROUND**：必须附原因说明

## 禁止

- 禁止残留被注释掉的代码
- 禁止装饰性分隔线（`// ===`、`// ---`）
- 禁止文件头部描述注释（如 `// xxx.ts - XXX 控制器`）
- 禁止注释明显代码（如 `// 加 1` 后跟 `i++`）
