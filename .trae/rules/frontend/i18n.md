# i18n 国际化规则

## 基本原则

- 所有用户可见的文本必须通过 i18n 翻译，不硬编码语言字符串
- 代码注释和变量命名使用中文，不与 UI 文本混淆

## Locale Code 规范

- 统一使用 BCP 47 格式：`zh-CN`、`en`、`ja`、`ko`（连字符，不是下划线）
- 该 code 必须与 `@nuxt/ui/locale` 中对应 locale 的 `code` 字段一致，否则 `ULocaleSelect` 无法正确匹配
- 新增语言时，先确认 `@nuxt/ui/locale` 中该 locale 的导出名和 `code` 值

## 翻译文件管理

- 翻译文件位于 `packages/i18n/locales/`，以 locale code 命名（如 `zh-CN.json`）
- 同步更新三处：
  1. `packages/i18n/locales/<code>.json` — 翻译内容
  2. `packages/i18n/src/index.ts` — import 和 export
  3. `packages/i18n/package.json` — exports 路径
  4. `apps/frontend/nuxt.config.ts` — i18n.locales 数组
- 翻译 key 使用嵌套结构，按功能模块分组（如 `common.submit`、`auth.login`）

## Locale 切换

- 通过 `useI18n().setLocale(code)` 切换语言
- `LocaleSelect.vue` 中使用 `ULocaleSelect` 组件，`modelValue` 绑定 `locale`，事件通过 `setLocale` 处理
- 切换后 `UApp` 的 `:locale` prop 会自动更新（通过 `app.vue` 中的 `locales[locale]` 绑定）
- 不要手动拼接 locale 对象或使用 `extendLocale` 做 hack，如有不匹配请修正 i18n 侧的 code

## 新增语言检查清单

新增语言时按以下顺序操作：

1. 在 `packages/i18n/locales/` 下创建 `<code>.json` 翻译文件
2. 在 `packages/i18n/src/index.ts` 中 import 并 export
3. 在 `packages/i18n/package.json` 的 exports 中添加路径映射
4. 在 `apps/frontend/nuxt.config.ts` 的 `i18n.locales` 数组中添加 `{ code, name, file }`
5. 在 `LocaleSelect.vue` 的 `availableLocales` 中添加对应 locale 对象
6. 确认 `@nuxt/ui/locale` 中存在该语言的导出（如 `import { zh_cn } from '@nuxt/ui/locale'`）
7. 验证：切换到新语言后，所有页面文本正确显示，`ULocaleSelect` 选中状态正确
