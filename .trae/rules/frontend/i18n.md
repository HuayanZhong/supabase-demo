> **MANDATORY** — 任意 .vue / .ts 文件中不得出现未被 `t()` 包裹的中英日韩显示文本。
> 违反此规则的改动将被 hook 自动阻断。

## 核心规则

- 所有显示文本通过 `$t('key')` 或 `useI18n()` 的 `t()` 函数读取，**禁止硬编码任何显示文本**
- 翻译 key 在 `@supabase/i18n` 包 (`packages/i18n/locales/`) 的对应语言文件中维护
- 新增文本时，**必须同步补充全部 4 种语言**（zh-CN / en / ja / ko），缺一不可
- 翻译 key 使用空格分隔的简洁命名：`AI EmptyTitle`、`Settings TabProfile`、`Month Jan`

## 常见违规模式

| ❌ 错误                   | ✅ 正确                                  |
| ------------------------- | ---------------------------------------- |
| `<span>首页</span>`       | `<span>{{ $t('Nav Home') }}</span>`      |
| `label: '设置'`           | `label: t('Settings Title')`             |
| `toast.success('已保存')` | `toast.success(t('Save Success'))`       |
| `placeholder="搜索..."`   | `:placeholder="t('Search')"`             |
| `<UTooltip text="主题">`  | `<UTooltip :text="t('Settings Theme')">` |
| `ref.value = '默认'`      | `ref.value = t('Default')`               |

## 硬编码检测规则（Regex）

hook 自动扫描以下模式，发现即阻断：

```
# .vue 文件中未被 t() 包裹的中文字符
[\u4e00-\u9fff]+      （中文）
[\u3040-\u309f]+      （日文平假名）
[\u30a0-\u30ff]+      （日文片假名）
[\uac00-\ud7af]+      （韩文）
```

**排除项**（不被判定为硬编码）：

- `t('...')` 或 `$t('...')` 内部的内容
- `<style>` / `<style scoped>` 标签内的内容（CSS）
- `import` 语句、路径字符串
- `//` 注释和 `/* */` 注释
- URL / 路由路径
- 环境变量名（`process.env.*`、`import.meta.env.*`）

## 修改 .vue / .ts 文件后的强制自检清单

每修改一个文件，必须逐条检查：

1. ☐ 搜索文件中是否有未被 `t()` 包裹的中文字符：`grep -n '[\u4e00-\u9fff]' path/to/file.vue | grep -v 't(' | grep -v '\$t(' | grep -v '//' | grep -v 'import'`
2. ☐ 确认新增的所有 key 已在 4 个 locale 文件中（zh-CN / en / ja / ko）全部定义
3. ☐ 确认未删除其他页面仍在使用的 key（`grep` 搜索该 key 在代码库中的引用）
4. ☐ 如果修改了已有 key 的值，确认该 key 在其他页面没有依赖
5. ☐ 如果修改了 locale JSON 文件，最后运行 `pnpm -F frontend check-types` 验证 JSON 格式和引用正确性

## 关键注意事项

- **insight / category / task 等数据类 key 不要删除**——它们可能被首页/其他未修改的文件引用
- 多个 Agent 并行修改 locale 文件时，后写入的 Agent 可能覆盖前面的新增 key → **避免并行修改同个 locale 文件**
- 用 `t()` 包裹的字符串值可以是 template 中的 props、placeholder、aria-label、toast 文本、下拉选项等所有用户可见文本
