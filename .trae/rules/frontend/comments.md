> **MANDATORY** — 所有前端 .vue 文件和 composable .ts 文件必须至少包含 1 条文件级注释，解释该文件的用途。

## 文件级注释（必须）

每个 `.vue` 文件和 composable `.ts` 文件顶部必须有一条注释说明组件/文件的用途。

**格式**：`// 一句话说明`（行内注释，不用 JSDoc）

```
// 日历与天气信息卡片            ← ✅ 合格
// AI 聊天输入框组件              ← ✅ 合格
// 最近对话列表卡片               ← ✅ 合格
```

```
<!-- 日历与天气信息卡片 -->         ← ❌ 不要用 HTML 注释，要用 script 中的 //
```

## Composable 注释要求

- 每个 `export function useXxx()` 上方必须有 JSDoc 或行内注释解释用途
- 高复杂度逻辑（超过 10 行）必须有"为什么这么做"的注释

```
// useChartTheme: 根据 Nuxt UI 主题模式返回 ECharts 配色方案
export function useChartTheme() { ... }
```

## 注释质量要求

- **只写"为什么"，不写"做了什么"**（代码本身应当能自说明做了什么）
- 好的注释：`// SSR：使用 useRequestFetch 确保 cookie 正确传播到内部 API 路由`
- 坏的注释：`// 获取当前用户信息`（下方 `getSession()` 已经说明）
- 坏的注释：`// 搜索关键词`（变量名 `searchQuery` 已经说明）
- 坏的注释：`// 登录表单字段`（模板中的 `v-model` 绑定已经说明）

## 常见违规模式

| ❌ 错误             | 问题                     | ✅ 正确处理            |
| ------------------- | ------------------------ | ---------------------- |
| `// 侧边栏导航菜单` | 变量名 `navItems` 已说明 | 删除                   |
| `/** 退出登录 */`   | JSDoc 用于内部函数       | 改为 `//` 或删除       |
| `<!-- 页面头部 -->` | HTML 注释冗余            | 删除                   |
| `// TODO: 修复`     | 缺少责任人               | `// TODO(owner): 修复` |
| `// 总工时`         | `totalHours` 已自说明    | 删除                   |
| 无注释              | 新建文件必须有一条       | 添加文件级说明         |

## 修改文件时的规则

- 修改已有文件时，**不要删除原有的**注释，除非确定是冗余"what"注释
- 如果文件已有<3条注释且全部是"what"注释，应补充至少1条"why"注释
- 不要为了"凑够注释数"而添加无意义注释
