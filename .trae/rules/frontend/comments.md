# 注释规范

## 通用规则

- 所有注释使用中文
- 注释写在需要解释的代码上方，不放在行尾
- 注释与代码之间保持一个空行（注释上方无空行要求，下方代码紧跟注释）

## 什么时候需要写注释

以下情况必须加注释：

- **非显而易见的业务逻辑** — 算法、条件判断、边界处理等一看不懂的代码
- **外部依赖的特殊用法** — 调用了有坑或非直觉的 API
- **临时方案 / TODO** — 标记为后续优化

## 什么时候不需要写注释

以下情况不允许注释：

- **自解释的代码** — 变量名、函数名已经清晰表达意图的，不加注释
- **显而易见的操作** — 如 `// 设置 loading 状态` 后面跟 `loading.value = true`
- **Vue 模板结构标记** — 如 `<!-- 顶栏 -->`、`<!-- 表单 -->`，用组件名和 HTML 语义标签本身就够了
- **重复代码内容的注释** — 注释不能是代码的翻译

## Vue 组件注释

- 组件文件顶部不加文件描述注释（文件名和组件名已经说明）
- `<script setup>` 内部的逻辑分块可以用单行注释分隔
- `<template>` 内原则上不加 HTML 注释，结构通过缩进和空行表达
- props / emits / slots 不加 JSDoc，类型定义本身就是文档

## TypeScript 注释

- 函数不加 JSDoc，除非是共享工具函数且行为非显而易见
- 类型定义不加注释，命名本身就是文档
- 复杂类型可在上方加简短说明
