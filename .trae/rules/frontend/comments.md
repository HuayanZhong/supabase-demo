# 注释规范

## 通用规则

- 所有注释使用中文
- 注释写在需要解释的代码上方，不放在行尾
- 注释与代码之间保持一个空行（注释上方无空行要求，下方代码紧跟注释）

## 必须加注释的场景

- **模块/区域说明** — 文件内的功能模块划分、页面区块划分，用注释标明区域
- **非显而易见的业务逻辑** — 算法、条件判断、边界处理等一眼看不懂的代码
- **外部依赖的特殊用法** — 调用了有坑或非直觉的 API
- **临时方案 / TODO** — 标记为后续优化
- **工具函数的功能说明** — 被多处调用的函数，简要说明其作用和入参

## 不允许注释的场景

- **显而易见的操作** — 如 `// 设置 loading 状态` 后面跟 `loading.value = true`，代码本身已经说明
- **重复代码内容的注释** — 注释不能是代码的翻译

## Vue 组件注释规范

### 模板注释

- 长页面用 `<!-- 区域名 -->` 标记区块边界，方便快速定位
- 短页面（一屏内）不需要模板注释，缩进和空行已经足够

### Script 注释

- `<script setup>` 内部的逻辑可以用单行注释分隔不同功能块
- 表单字段等声明式数据不加注释，类型本身就是文档

### 不允许的

- 组件文件顶部不加文件描述注释（文件名和组件名已经说明）
- props / emits / slots 不加 JSDoc
- 模板内不加功能描述注释（如 `<!-- 这里显示用户名 -->`），可见即所得

## TypeScript 注释规范

- 通用工具函数加 `/** 说明 */` JSDoc，简要说明用途和参数
- 类型定义不加注释，命名本身就是文档
- 复杂类型可在上方加简短说明

## 注释风格

- 模板内用 `<!-- 注释内容 -->`
- TypeScript 代码中用 `// 注释内容`（单行）或 `/** 注释内容 */`（JSDoc）
- 不写大段注释，保持简洁
