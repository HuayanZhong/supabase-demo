> 每次修改前端代码后，必须按顺序执行以下检查。

| #   | 检查项     | 命令                           | 通过标准                    |
| --- | ---------- | ------------------------------ | --------------------------- |
| 1   | 类型检查   | `pnpm -F frontend check-types` | exit code 0                 |
| 2   | Lint 检查  | `pnpm -F frontend lint`        | 0 errors（warnings 可接受） |
| 3   | 浏览器验证 | 打开对应页面确认渲染           | 无控制台错误、页面不空白    |

## 常见问题排查

| 症状                  | 可能原因                                               |
| --------------------- | ------------------------------------------------------ |
| 页面空白 / 组件不渲染 | 组件名未匹配 Nuxt 自动导入规则                         |
| 控制台 null/undefined | 数据在 `ClientOnly` 之前被访问，加 `v-if` 守卫         |
| 水平滚动条            | 元素有负 margin 或 blur 滤镜溢出，加 `overflow-hidden` |
| i18n 不生效           | key 不在 locale 文件中，或 `t()` 参数名不一致          |
