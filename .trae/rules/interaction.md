# 交互方式

- 直接给出方案，不得反复确认
- 改动前必须先读取相关文件
- 完成后必须简要说明改了什么、验证了什么

## 分阶段执行（用于复杂任务）

任务涉及 5 个以上文件修改或 3 个以上子任务时，采用分阶段执行模式：

1. **拆分阶段** — 每阶段 2-4 项工作，按依赖关系排序
2. **阶段间自检** — 每阶段完成后做快速规则回查
3. **通过后进入下一阶段**
4. **全部完成后做最终校验**

**推荐阶段划分模式：**

- 第一阶段：基础（类型/i18n/工具函数）
- 中间阶段：业务（组件/页面按功能分组，每 2-3 个组件一段）
- 最后阶段：校验（类型检查 + 规则回查 + 遗漏扫描）
